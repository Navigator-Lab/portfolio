Linux / DevOps Platform — Featured

NaviOps — Linux & NOC Operations Platform

A self-built Linux operations platform: a 28-module structured curriculum covering Linux administration, automation, networking, and security operations — with hands-on scripts and artifacts committed incrementally as I build, on a live system.

Scope
Linux fundamentals (filesystems, permissions, users/groups, processes, systemd, journald), Bash automation, networking (DNS/DHCP/NAT/firewalls), and security fundamentals (hardening, log analysis) — with Docker, AWS (EC2, S3, IAM, VPC), infrastructure-as-code (Ansible, Terraform), and security monitoring (Wazuh) covered across the curriculum, hands-on artifacts in progress.
What Makes It Different
Every module is taught through an attacker & defender lens — how an attacker abuses a feature (GTFOBins / MITRE ATT&CK techniques) and how a defender detects and hardens against it. Backed by a bank of 7 RHCSA-aligned troubleshooting drills (SSH lockout, broken fstab, failed systemd, broken sudo, full disk, SELinux, firewall) — direct prep for "walk me through how you'd debug X" interviews.
How It's Built
Each module follows the same build pattern: plan → real implementation → verification → documentation. I write every script and config myself; commit history reflects real, incremental progress on a live system.
Capstone
A curriculum capstone — an AWS + Terraform infrastructure project and an incident-response scenario, aligned with RHCSA-level Linux administration skills (in progress).
Repo Highlights
Real, runnable scripts and docs in the repo — e.g. scripts/git-health-check.sh, scripts/pre-commit-gitleaks.sh, and an architecture diagram in the README.
